# SEO engine plan

## Lessons applied from legacy

1. **Canonical URL** must match sitemap `loc` and internal links.
2. **One metadata pipeline** — Next.js `generateMetadata` per route; no duplicate `<title>` sources.
3. **Structured data:**
   - `Organization` + `WebSite` on home
   - `NewsArticle` or `Article` for news
   - `Event` for events
   - `JobPosting` for jobs (when fields complete)
   - `BreadcrumbList` when hierarchy exists
4. **Avoid invalid schema** — conditional blocks only when required fields exist (legacy used layered checks for sports; prefer **data-driven** schema builders).
5. **GA4:** measurement ID via env; separate numeric property ID for Data API — never confuse the two.

## Implementation

- **ISR / caching:** Use `revalidate` on public pages; `revalidatePath` / `revalidateTag` on publish.
- **Sitemaps:** Single index pattern via `sitemap.ts` returning multiple sitemaps if needed (Next 13+ supports array of sitemap entries or split files — follow Next docs for chosen version).
- **Search Console:** Manual property verification on Vercel; submit sitemap after launch.

## Anti-patterns to avoid

- Default meta keywords stuffed with legacy locality names.
- Thin autogenerated hub pages without unique copy.

## Chennai jobs SERP (ops checklist)

- **Canonical hub:** `/chennai-jobs` (legacy `/jobs` → 301). Sitemap and internal links must match.
- **Google Search Console:** Track query clusters (`chennai jobs`, `jobs in chennai`, role long-tails, `mychennaicity jobs`). Watch **Enhancements → Job postings** when eligible. Submit sitemap after deploy.
- **Rich Results Test:** Spot-check one open job detail URL after schema or template changes.
- **Seed / staging:** `npm run db:seed:chennai-jobs` loads sample open listings when `DATABASE_URL` is set.
- **Digital PR (manual):** Earn links from chambers, colleges, startup communities, and Chennai IT groups to `/guides/chennai-tech-careers` and `/chennai-jobs`. See [`CHENNAI_JOBS_OUTREACH.md`](CHENNAI_JOBS_OUTREACH.md) for angles and targets.
- **GBP / NAP:** If you have a physical office, keep Google Business Profile aligned with optional env: `NEXT_PUBLIC_ORG_ADDRESS_LOCALITY`, `NEXT_PUBLIC_ORG_STREET_ADDRESS`, `NEXT_PUBLIC_ORG_ADDRESS_REGION`, `NEXT_PUBLIC_ORG_POSTAL_CODE`, `NEXT_PUBLIC_ORG_ADDRESS_COUNTRY` (feeds Organization `address` on the home JSON-LD graph).

### GSC: property, sitemap, and KPIs (repo + operator)

| Step | What to do |
|------|------------|
| 1. Verify | Add the site in [Google Search Console](https://search.google.com/search-console) (URL-prefix or domain property). Optional: set `NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION` in Vercel so Next.js emits the `<meta name="google-site-verification">` tag (see `.env.example` / `AGENTS.md`). |
| 2. Submit sitemap | After deploy, submit `https://mychennaicity.in/sitemap.xml` in GSC → Sitemaps. For automation with a service account, configure env vars in `.env.example` and run `npm run gsc:submit-sitemap` (`scripts/gsc-submit-sitemap.ts`). |
| 3. Queries to watch | Head: `chennai jobs`, `jobs in chennai`, `chennai job openings`. Long-tail: role + Chennai (e.g. `react developer chennai`). Branded: `mychennaicity`, `mychennaicity jobs`. |
| 4. Enhancements | **Job postings** — fix errors/warnings when the report appears; correlate with Rich Results Test on live job URLs. |
| 5. Cadence | Monthly: impressions/clicks for jobs hub + 2–3 sample job URLs + guide URL; note any crawl anomalies. |

### Internal links (jobs hub)

Contextual inlinks now include home cross-nav, news hub (sidebar + empty state), events, directory, about, contact, editorial standards, glossary, economy topic, areas, and the careers guide. **Quarterly:** in GSC → Pages → filter URLs with few or no referring internal links and add links from related hubs or articles.
